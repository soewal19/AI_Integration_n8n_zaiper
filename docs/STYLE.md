# Стиль программирования (MCP Context7 Guidance)

Этот проект следует практикам Clean Architecture и стандартам Python‑разработки. Ниже — выжимка рекомендаций и конкретные правила в духе MCP Context7.

## Базовые правила
- PEP 8: длина строки ≤ 100 символов, осмысленные имена, пробелы вокруг операторов.
- Импорты: stdlib → сторонние → локальные; группировать блоками, сортировать по алфавиту.
- Аннотации типов: обязательны в публичных функциях/методах и у переменных модулей.
- Исключения не глотать; логировать с уровнем и контекстом.
- Логирование через `logging` с единым форматом и `LOG_LEVEL` из окружения.

## Архитектура и зависимости
- Слои: Domain ↔ Application ↔ Infrastructure ↔ Presentation.
- Зависимости направлены внутрь; композиция реализаций — в Presentation (CLI).
- Внедрение зависимостей через конструкторы (DIP).

## Докстринги и публичные API
- Формат Google/PEP257; краткое описание первой строкой.
- Публичные методы сервисов и интерфейсы документируются.

## Обработка ошибок
- Для ожидаемых ошибок использовать специфичные исключения; для сетевых — логировать причину и таймаут.
- В циклах и параллельных задачах — окружать try/except на границе, возвращать структурированный результат.

## Конфигурация
- 12‑Factor: все чувствительные данные и пути — из env; локальная подгрузка `config/settings.env`.
- Константы/пороговые значения — централизованы в одном месте (env/конфиг).

## Тестирование
- E2E — в директории `tests/`, имена `test_*.py`.
- Допускается код возврата 1, если есть фейлы эндпоинтов; тест проверяет артефакты (JSON‑отчёт).

## Конкурентность
- Для IO — `ThreadPoolExecutor` (или `async` при переходе на httpx).
- Таймауты обязательны; никогда не делать бесконечные ожидания.

## CI/CD и линтинг
- Ruff — базовый линтер/формат: ошибки E,F,I,UP; длина строки 100.
- Mypy — статическая проверка (мягкий режим, ignore missing imports).
- GitHub Actions: lint + tests + docker build.

## Безопасность
- Никогда не хранить токены в репозитории и логах CI; использовать Secrets/Env.
- В Docker — без лишних зависимостей; не писать байткод, буферизацию отключать.

## Примеры
- Конфиги линтеров и mypy — в `pyproject.toml`.
- .editorconfig — единые отступы и кодировки.
