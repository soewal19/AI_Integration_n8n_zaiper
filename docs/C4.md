# C4-модель: Мониторинг внешних API

## Контекст (C1)
```mermaid
graph TD
  Team[Команда / Airflow] -->|запуск| Service[Monitoring Service]
  Service -->|HTTP| External1[JSONPlaceholder Users/Posts/Comments]
  Service -->|Webhook| Slack[Slack Channel]
  Service -->|SQLite| DB[(health_history.db)]
```

## Контейнеры (C2)
```mermaid
graph TD
  subgraph Host[Хост / Docker контейнер]
    CLI[CLI (monitor.py)]
    App[Application Service]
    Checker[HTTP Checker (ThreadPool)]
    Notifier[Notifier (Slack/Console)]
    Repo[SQLite Repository]
    File[health_report.json]
  end

  CLI --> App
  App --> Checker
  App --> Notifier
  App --> Repo
  App --> File
```

## Компоненты (C3)
```mermaid
graph TD
  subgraph Domain
    Entities[entities.py: Endpoint, CheckResult, HealthReport]
  end
  subgraph Application
    Service[MonitoringService]
    Interfaces[interfaces.py: HTTPChecker, Notifier, ResultsRepository]
  end
  subgraph Infrastructure
    HTTP[http_client.py: ParallelHTTPChecker]
    Slack[notifiers.py: SlackOrConsoleNotifier]
    SQLite[database.py: SQLiteResultsRepository]
    Config[config.py: endpoints.json loader]
  end
  subgraph Presentation
    CLI[cli.py]
  end

  CLI --> Service
  Service --> Interfaces
  Service --> HTTP
  Service --> Slack
  Service --> SQLite
  Service --> Config
  Interfaces -.-> Entities
  HTTP -.-> Entities
  Slack -.-> Entities
  SQLite -.-> Entities
```

## Замечания
- Зависимости направлены внутрь: Application зависит от абстракций, а не реализаций.
- Компоненты можно подменять (например, TelegramNotifier, PostgresRepo) без изменения бизнес-логики.
